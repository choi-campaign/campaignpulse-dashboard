# CampaignPulse Collection Log Schema

최종 점검일: 2026-06-19

## 목적

`collection_log`는 데이터 수집 시도와 결과를 기록하는 운영 로그다.

- 데이터 최신성 판단
- 최근 수집 성공·실패 시각 확인
- 실패 원인 안내
- 매체별 최근 30일 수집 성공률 계산
- 광고주 수가 늘어날 때 수집 장애 추적

광고센터 진입만으로는 성공으로 기록하지 않는다. 리포트 파일 또는 API 성과 행을 실제로 확보한 경우에만 `success`로 기록한다.

## 저장 위치

- 기본 DB: `data/storage/collection_log.sqlite3`
- 초기화 확인: `python -m aimaos.storage.collection_log`

DB 파일이 아직 없을 때 상태 조회 함수는 빈 결과를 반환하며, 조회만으로 새 DB를 만들지 않는다.

## 테이블

테이블명: `collection_log`

| 컬럼 | 설명 | 예시 |
|---|---|---|
| collection_id | 수집 시도 고유 ID | `naver_20260619_031200` |
| advertiser_id | 광고주 또는 계정 식별자 | `sample_client_001` |
| media | 매체 키 | `naver`, `gmarket`, `auction` |
| started_at | 수집 시작 시각 | `2026-06-19 03:12:00` |
| finished_at | 수집 종료 시각 | `2026-06-19 03:14:00` |
| status | 결과 상태 | `success`, `failed`, `partial`, `no_data` |
| rows_collected | 실제 수집 행 수 | `1520` |
| file_count | 생성 또는 감지한 파일 수 | `3` |
| storage_used_mb | 이번 수집에 추가된 용량 | `12.4` |
| error_code | 실패 코드 | `LOGIN_REQUIRED` |
| error_message | 운영자가 이해할 수 있는 실패 사유 | `로그인 인증이 필요합니다.` |

## 상태 기준

| 상태 | 의미 | 화면 처리 |
|---|---|---|
| `success` | 실제 데이터 수집 성공 | 마지막 수집 시각과 성공률 갱신 |
| `partial` | 일부 데이터만 수집 | `주의` 표시와 확인 사유 안내 |
| `no_data` | 호출은 완료했으나 성과 행 또는 파일 없음 | `연결 안됨`과 최신 성과 확인 안내 |
| `failed` | 수집 실패 | `연결 안됨`과 실제 실패 사유 안내 |

## Data Status Center 연결

`collection_status_by_media()`는 매체별로 다음 값을 계산한다.

- 가장 최근 시도 상태와 종료 시각
- 마지막 성공 시각
- 마지막 실패·부분 성공·데이터 없음 시각
- 가장 최근 오류 코드와 설명
- 최근 30일 성공 횟수, 전체 시도 횟수, 성공률

시간대가 포함된 ISO 시각은 한국 표준시 기준의 비교 가능한 시각으로 정규화한다. 최근 시도 상태가 실패여도 과거 마지막 성공 시각은 보존한다.

종합 대시보드의 데이터 상태 요약은 파일 수정 시각과 수집 로그 시각 중 더 최근 값을 사용한다. 매체별 상태는 실제 수집 로그를 우선 반영한다.

## 운영 원칙

- 광고주 비밀번호를 저장하지 않는다.
- 캡차 또는 2차 인증을 우회하지 않는다.
- 실제 광고주 데이터와 API 키를 저장소에 올리지 않는다.
- 실패 메시지는 `인증 필요`, `리포트 확인 필요`, `데이터 없음`처럼 운영자가 이해할 수 있게 기록한다.
- G마켓·옥션은 광고센터 진입 성공과 리포트 수집 성공을 구분한다.
