# Channel Intelligence Layer

## 1. 목적

Channel Intelligence Layer는 AIMAOS를 단순 광고매체 분석기가 아니라 통합 마케팅 운영 관제 플랫폼으로 확장하기 위한 상위 구조입니다.

관리 대상은 광고매체, 오픈마켓, 커머스 플랫폼, 글로벌 채널입니다.

현재 단계에서는 실제 외부 데이터 수집, 로그인 자동화, 크롤링을 구현하지 않습니다. 먼저 채널 등록 구조, 통합 KPI 기준, 벤치마크 기준, 관리자 관리 구조를 만듭니다.

## 2. 채널 영역

| 영역 | 목적 | 대표 채널 |
| --- | --- | --- |
| 광고매체 | 광고 집행 성과 분석 | 네이버 검색광고, 네이버 쇼핑광고, 네이버 브랜드검색, 카카오 광고, 구글 광고, 메타 광고, 틱톡 광고, 유튜브 광고, 당근 광고 |
| 오픈마켓 | 판매채널 성과 분석 | 쿠팡, G마켓, 옥션, 11번가 |
| 커머스 플랫폼 | 신규 성장 채널 분석 | 스마트스토어, SSG닷컴, 롯데온, 컬리, 오늘의집, 무신사, 토스쇼핑 |
| 글로벌 채널 | 해외 판매 및 신흥 채널 분석 | 알리익스프레스, 테무, 아마존, 쇼피, 큐텐 |

## 3. DB 기반 채널 관리

채널은 화면 코드에 직접 고정하지 않고 SQLite DB에서 관리합니다.

현재 DB 위치:

```text
data/channel_registry.sqlite
```

테이블:

```text
channels
```

핵심 컬럼:

| 컬럼 | 의미 |
| --- | --- |
| channel_name | 채널명 |
| channel_group | 광고매체, 오픈마켓, 커머스 플랫폼, 글로벌 채널 |
| purpose | 채널 관리 목적 |
| metrics | 분석 항목 |
| brand_color | 화면 표시 색상 |
| is_active | 활성/비활성 상태 |
| display_order | 화면 표시 순서 |

## 4. 통합 KPI

통합 관제 화면은 아래 KPI를 기준으로 설계합니다.

- 총 매출
- 총 광고비
- 총 주문수
- 총 전환수
- 총 ROAS
- 총 신규고객
- 총 재구매율
- 총 객단가
- 총 CAC
- 총 LTV

## 5. 벤치마크 엔진 방향

향후 채널별 데이터가 누적되면 아래 기준으로 벤치마크를 생성합니다.

- 채널별 평균 CPC
- 채널별 평균 CTR
- 채널별 평균 ROAS
- 채널별 평균 전환율
- 채널별 평균 객단가
- 채널별 성장률

업종별 벤치마크는 별도 계층에서 연결합니다.

- 업종 평균 CPC
- 업종 평균 CTR
- 업종 평균 ROAS
- 업종 평균 전환율
- 업종 평균 객단가
- 업종 평균 광고비

## 6. AI 운영 점검 방향

AI 운영 점검은 채널 간 상대 성과를 비교합니다.

예시:

- 특정 채널의 ROAS가 전체 평균보다 낮은지 확인
- 광고비 비중이 높은데 매출 기여가 낮은 채널을 표시
- 고효율 채널 확대 후보를 제안
- 저효율 채널 예산 축소 또는 운영 점검을 제안

## 7. 현재 구현 범위

현재 구현된 항목:

- Streamlit 메뉴 `채널 통합 관제`
- Streamlit 메뉴 `채널 관리자`
- SQLite 기반 채널 레지스트리
- 기본 채널 자동 등록
- 채널 추가
- 채널 수정
- 채널 활성/비활성 관리
- 샘플 통합 KPI 대시보드
- 샘플 채널 벤치마크
- 샘플 AI 운영 점검 문구

현재 구현하지 않는 항목:

- 실제 외부 채널 로그인
- 실제 크롤링
- 실제 API 수집
- 실제 주문/매출 데이터 자동 수집
- 캡차 우회 또는 보안 우회
