## 8. legacy / next 구분 필요 여부

구분이 필요합니다.

현재 사용자가 설명한 G마켓/옥션 광고센터 리포트는 기존 광고센터 기준이므로 `legacy`로 분류합니다. 새 광고센터가 나오면 컬럼명, 시트 구조, 기간 기준, 전환/매출 기준이 바뀔 수 있으므로 `next` profile을 별도로 만들어야 합니다.

## 9. 기존 MVP 보존 여부

보존했습니다.

이번 작업은 컬럼 alias 확장과 검증 리포트 보강입니다. 기존 광고 성과 계산식, 분석 엔진, 보고서 계산값, Rule Engine 계산 기준은 변경하지 않았습니다.

## 10. 실제 파일 검증 우회 방식

J: 드라이브 직접 접근이 막힌 문제는 Streamlit 광고 분석 화면의 `원본 파일 진단` 기능으로 우회합니다.

사용 방식:

1. Streamlit 앱에서 `광고 분석` 메뉴로 이동
2. `원본 파일 진단` 영역에 G마켓/옥션/네이버/11번가 원본 파일 업로드
3. 원본 저장이 필요하면 `진단 후 원본 파일을 data/raw/uploaded_diagnostics 폴더에 보관` 체크
4. 추정 매체, 적용 profile, 읽은 컬럼명, 매핑 성공/실패, 판단 불가 지표, 오늘 해야 할 일 생성 가능 여부 확인

이 기능은 진단만 실행하며 기존 분석 계산 로직과 보고서 생성 결과에는 영향을 주지 않습니다.
