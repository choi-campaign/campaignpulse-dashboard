# CampaignPulse UX Audit

작성일: 2026-06-13

## 1. 사이드바 UI/UX 문제

- 기존 사이드바는 메뉴 버튼의 폭, 높이, 배지 위치가 균일하지 않아 개발 중 화면처럼 보일 수 있었다.
- 메뉴명과 숫자 배지가 한 덩어리처럼 보여서 현재 메뉴와 운영 수치를 빠르게 구분하기 어려웠다.
- 하단 브리핑은 수치만 보여주고 실제 이동 행동으로 이어지지 않았다.
- 첫 메뉴가 데이터 상태 중심으로 보일 경우, 전체 운영 요약 화면이라는 역할이 약해질 수 있었다.

## 2. 수정한 내용

- 메뉴 구조를 `종합 대시보드` 중심으로 재정리했다.
- 사이드바 메뉴 버튼을 실제 `st.button` 기반으로 구성했다.
- 선택된 메뉴는 `st.session_state["active_page"]`에 저장하고, 클릭 시 `st.rerun()`으로 해당 화면만 다시 그리도록 정리했다.
- 메뉴 번호와 메뉴명을 왼쪽에 두고, 숫자 배지는 오른쪽 별도 영역으로 분리했다.
- 사이드바 버튼 CSS는 `[data-testid="stSidebar"] .stButton > button` 범위에만 적용했다.
- 하단 `오늘 브리핑` 아래에 실제 이동 버튼을 추가했다.
- 메인 버튼, 업로드 버튼, 다운로드 버튼에는 사이드바 버튼 스타일이 적용되지 않도록 범위를 제한했다.

## 3. 종합 대시보드 역할 정의

`종합 대시보드`는 데이터 상태만 보는 화면이 아니라 캠페인펄스의 전체 운영 상태를 요약하는 첫 화면이다.

포함 범위:

- 데이터 상태
- 우선 처리 이슈
- 광고 성과 흐름
- 보고서 상태
- 채널 연결 상태

사용자는 종합 대시보드에서 전체 흐름을 확인하고, 자세한 업무는 각 메뉴로 이동한다.

## 4. 오늘 브리핑 클릭 이동 구조

사이드바 하단 빠른 이동 버튼:

- `운영 현황 보기` -> `운영 현황`
- `이슈 확인` -> `우선 처리 이슈`
- `보고서 센터` -> `보고서 센터`

이동 방식:

- `st.sidebar.button`
- `st.session_state["active_page"]`
- `st.rerun()`

HTML 카드만 클릭 가능한 것처럼 보이게 만드는 방식은 사용하지 않았다.

## 5. 메뉴 클릭 딜레이 평가 결과

평가 결과:

- 전체 페이지를 동시에 렌더링하는 구조가 아니라 현재 선택된 메뉴만 렌더링한다.
- 체험 데이터 분석 스냅샷은 `load_sample_analysis_snapshot()`에서 `st.cache_data`를 사용하고 있다.
- 해당 캐시는 파일 쓰기, 보고서 생성, DB 쓰기와 연결되지 않은 순수 로딩/요약 범위다.
- 보고서 생성, 업로드 파싱, DB 쓰기, 채널 저장 로직에는 캐시를 추가하지 않았다.
- 종합 대시보드에서 보고서 목록과 채널 상태를 요약 조회하므로, 보고서 파일이 매우 많아질 경우 별도 경량 인덱스가 필요할 수 있다.

현재 판단:

- 메뉴 클릭 지연을 만드는 치명적인 구조는 발견하지 못했다.
- 대량 광고주 운영 단계에서는 보고서 목록, 원본 파일 목록, 채널 상태 요약의 별도 저장 테이블 또는 캐시 정책이 필요하다.

## 6. 05 쇼핑몰/매체 분석 현재 구현 상태

현재 구현 상태:

- `쇼핑몰/매체 분석` 메뉴는 확장 준비 화면으로 구현되어 있다.
- 스마트스토어, 쿠팡, G마켓, 옥션, 11번가를 대상 플랫폼으로 안내한다.
- 예시 데이터가 있으면 상위 50행 미리보기와 예시 지표를 보여준다.
- 실제 크롤러, 로그인 자동화, 외부 데이터 수집은 구현되어 있지 않다.

정상 작동 여부:

- 현재 범위에서는 정상이다.
- 실제 데이터 수집 기능으로 오해하지 않도록 “운영 준비 화면” 문구를 유지한다.

부족한 점:

- 실제 쇼핑몰별 성과 파일 연결은 아직 없다.
- 상품 단위 상세 분석 이동은 아직 없다.
- 광고 성과와 상품 경쟁력 연결은 설계 단계다.

바로 수정한 내용:

- 메뉴명을 `쇼핑몰/매체 분석`으로 정리했다.
- 화면 목적 문구를 쇼핑몰과 오픈마켓 통합 관점으로 정리했다.

추가 개발 필요:

- 쇼핑몰/매체별 업로드 또는 수집 데이터 연결
- 상품 단위 성과 요약
- 채널별 매출/주문/광고비 기준 통합 스키마
- 상품 상세 점검 화면

## 7. 06 보고서 센터 현재 구현 상태

현재 구현 상태:

- 최근 보고서 목록을 조회한다.
- 대상별 보고서 유형을 선택할 수 있다.
- 맞춤 요약 TXT, Excel, Markdown, TXT 다운로드가 가능하다.
- 보고서 파일명에는 캠페인펄스 기준을 적용한다.

정상 작동 여부:

- 생성된 보고서가 있으면 목록과 다운로드 버튼이 표시된다.
- 생성된 보고서가 없으면 광고 성과 분석 메뉴에서 먼저 분석하라는 안내가 표시된다.

부족한 점:

- 보고서 생성 예정 상태, 검토 상태, 제출 완료 상태는 아직 분리되어 있지 않다.
- 보고서 표지 브랜딩과 광고주 제출 흐름은 추가 설계가 필요하다.

바로 수정한 내용:

- 메뉴명을 `보고서 센터`로 유지하고, 보고서 제출물 관점의 문구를 유지했다.
- 우선 처리 이슈 반영 후보 문구로 정리했다.

추가 개발 필요:

- 보고서 상태값 추가
- 제출용 표지 템플릿
- 광고주별 보고서 이력 관리
- 보고서 예약/검토 워크플로

## 8. 07 채널 관리 현재 구현 상태

현재 구현 상태:

- `통합 관제`와 `채널 관리자` 탭으로 구성되어 있다.
- 채널 등록, 수정, 활성/비활성 상태를 관리할 수 있다.
- 채널 그룹과 대표 색상, 분석 항목을 관리한다.
- 통합 관제 화면에서는 예시 성과와 채널 운영 브리핑을 보여준다.

정상 작동 여부:

- 현재 DB 구조 기준으로 채널 목록 조회와 수정 폼은 동작한다.
- 실제 매체 연결 상태와 수집 성공률은 일부 별도 수집 로그 구조와 연결이 필요하다.

부족한 점:

- 채널 연결 성공/실패 이력과 완전히 연결되어 있지는 않다.
- 매체별 컬럼 매핑 상태를 운영자 관점으로 보여주는 화면은 아직 부족하다.

바로 수정한 내용:

- 사이드바 메뉴명과 배지 구조를 정리했다.
- 종합 대시보드에서 채널 관리로 이동할 수 있는 카드와 버튼을 제공했다.

추가 개발 필요:

- 수집 로그 기반 연결 상태 표시
- 매체별 최근 성공/실패 원인 표시
- 컬럼 매핑 진단 결과와 채널 관리 연결
- 채널별 수집 방식 안내

## 9. 08 설정 현재 구현 상태

현재 구현 상태:

- 보호 중인 핵심 기능 목록을 보여준다.
- 체험 데이터, 채널 관리 데이터, 계정 연결 상태, 핵심 기능 보호 원칙을 표시한다.
- 데이터 수집 구조와 대용량 처리 원칙, 운영 개선 원칙을 안내한다.

정상 작동 여부:

- 현재는 정보 확인용 화면으로 정상이다.
- 위험한 설정 저장이나 분석 기준 변경 기능은 제공하지 않는다.

부족한 점:

- 실제 사용자가 변경 가능한 설정은 제한적이다.
- 브랜드 설정, 보고서 설정, 운영 기준 설정은 아직 별도 저장 구조가 없다.

바로 수정한 내용:

- 기존 보호 대상 기능을 유지했다.
- 개발자용 표현보다 운영자 관점 문구를 우선했다.

추가 개발 필요:

- 브랜드/보고서/운영 기준 설정 분리
- 설정 변경 이력 기록
- 위험 설정 변경 시 검토 단계 추가

## 10. 바로 수정한 항목

- 첫 메뉴를 `종합 대시보드`로 변경
- `우선 처리 이슈`, `광고 성과 분석`, `쇼핑몰/매체 분석` 메뉴명 정리
- 사이드바 메뉴 버튼을 실제 클릭 가능한 `st.button` 구조로 변경
- 사이드바 배지를 메뉴 오른쪽으로 분리
- 선택 메뉴 강조 상태 적용
- 하단 빠른 이동 버튼 추가
- 종합 대시보드에 전체 요약 카드와 상세 메뉴 이동 버튼 추가
- 사이드바 CSS 범위 제한
- 보이지 않는 보조 화면의 제목을 운영자 관점 문구로 정리

## 11. 추가 개발이 필요한 항목

- 대량 보고서 목록 조회 최적화
- 광고주별 데이터 상태 인덱스
- 쇼핑몰/매체 실제 데이터 연결
- 보고서 상태 워크플로
- 채널 연결 성공/실패 이력과 채널 관리 화면 연결
- 설정 저장 구조 설계

## 12. 기능 로직 미변경 원칙

기존 기능 로직은 변경하지 않았다.

변경하지 않은 항목:

- 광고 성과 계산 로직
- Rule Engine 계산 기준
- 데이터 업로드 파싱 로직
- 컬럼 매핑 로직
- DB 스키마
- 보고서 계산 기준
- 기존 분석 기준
- 테스트 데이터 구조
- 기존 데이터 처리 방식
- 보고서 산출물 계산 로직
- 채널 관리 DB 구조
