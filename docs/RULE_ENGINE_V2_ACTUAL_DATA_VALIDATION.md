# Rule Engine V2 실제 광고주 데이터 검증 계획

작성일: 2026년 6월 8일

## 1. 현재 상태

새 기능 개발은 보류합니다.

이번 단계의 목표는 오늘 해야 할 일 엔진이 실제 광고 운영자에게 도움이 되는지 검증하는 것입니다.

중요: 현재 `data/raw` 폴더에는 실제 광고주 원본 파일 3~5개가 아직 없습니다.

현재 확인된 파일:

```text
data/raw/sample_ads.csv
data/raw/test_ads_full_visible.csv
data/raw/gmarket_auction/sample_advertiser/README.md
```

위 파일은 샘플 또는 보관 안내 파일이므로, 실제 광고주 데이터 검증 완료로 볼 수 없습니다.

## 2. 실제 검증 조건

실제 검증에는 아래 조건을 만족하는 광고주 데이터 3~5개가 필요합니다.

- 서로 다른 업종보다 서로 다른 운영 상태를 우선 확보
- 실제 광고센터에서 내려받은 CSV/XLSX/XLS 원본
- 최근 30일 이상 또는 최근 90일 기준 권장
- 날짜, 매체, 캠페인, 광고그룹, 상품명 또는 키워드, 노출수, 클릭수, 광고비, 전환수, 매출 중 최대한 많은 컬럼 포함
- 가능하면 G마켓, 옥션, 네이버, 11번가 중 2개 이상 매체 포함

검증 대상 광고주는 최소 아래 상태를 포함해야 합니다.

| 상태 유형 | 검증 목적 | Rule Engine이 구분해야 하는 핵심 |
| --- | --- | --- |
| 성과 우수 광고주 | 확대 후보를 과하지 않게 찾는지 확인 | ROAS, 전환수, 광고비 규모가 함께 양호한지 |
| 성과 부진 광고주 | 축소 또는 구조 점검 대상을 찾는지 확인 | 광고비 대비 매출과 전환이 낮은지 |
| 전환 문제 광고주 | 광고 문제와 추적/상품 문제를 단정 없이 구분하는지 확인 | 클릭은 있으나 전환이 낮거나 없는지 |
| 예산 부족 광고주 | 단순 저성과가 아니라 예산 제약 상황을 구분하는지 확인 | 노출/클릭 기회가 있는데 광고비 규모가 너무 작은지 |
| 시즌성 영향 광고주 | 일시적 하락과 구조적 문제를 구분하는지 확인 | 특정 기간 변동이 계절/행사 영향일 가능성이 있는지 |

단순히 업종만 다른 데이터는 검증 효율이 낮습니다. V2 검증에서는 업종 다양성보다 상태 다양성을 우선합니다.

파일 위치:

```text
data/raw/actual_validation/
```

또는 Streamlit `광고 분석 > 원본 파일 진단` 화면에서 직접 업로드합니다.

## 3. 검증 작업 순서

1. 실제 광고주 파일 3~5개 확보
2. 원본 파일 진단 실행
3. 기존 광고 분석 실행
4. 생성된 오늘 해야 할 일 이슈 전부 출력
5. 각 이슈를 `유용함`, `보통`, `불필요`로 분류
6. 잘못된 추천 수집
7. 부족한 Rule 수집
8. 광고주가 물어볼 질문 유형 수집
9. 광고주 설명 문구 품질 평가
10. Rule Engine V2 설계 확정

## 7. 잘못된 추천 수집 기준

아래 유형은 V2에서 반드시 줄여야 합니다.

- 전환 추적 누락 가능성이 있는데 광고 성과 문제로 단정하는 경우
- 품절, 가격 변경, 리뷰 악화 등 상품 요인을 모르면서 광고 문제로 단정하는 경우
- 예산 확대 근거가 약한데 확대를 권하는 경우
- ROAS는 높지만 전환 수가 너무 적은데 확대 후보로 보는 경우
- 신규 캠페인 학습 기간인데 성과 저조로 과하게 판단하는 경우
- 브랜드 키워드와 일반 키워드를 같은 기준으로 평가하는 경우
- 매체별 특성을 무시하고 동일 기준으로 판단하는 경우

## 8. 부족한 Rule 후보

실제 데이터 검증 전 기준으로 예상되는 부족 Rule입니다.

| Rule 후보 | 필요한 이유 | 필요한 데이터 |
| --- | --- | --- |
| 전환 추적 의심 Rule | 클릭과 매출 흐름이 비정상일 때 광고 문제가 아닐 수 있음 | 클릭수, 전환수, 매출, 전환 추적 상태 |
| 저전환 고비용 Rule | 광고비는 큰데 전환이 낮은 항목 우선 점검 | 광고비, 전환수, CPA, ROAS |
| 고ROAS 저볼륨 Rule | ROAS만 높고 전환 수가 적은 확대 오판 방지 | ROAS, 전환수, 광고비 |
| 브랜드/일반 키워드 분리 Rule | 브랜드 키워드는 CTR/ROAS 기준이 일반 키워드와 다름 | 캠페인명, 키워드명, 브랜드 여부 |
| 상품 품절 의심 Rule | 광고비가 나가는데 매출이 갑자기 줄 때 상품 상태 확인 필요 | 상품명, 클릭수, 전환수, 매출, 재고 데이터 |
| CPC 급등 Rule | 입찰 경쟁 심화 또는 자동입찰 이상 가능성 확인 | CPC 기간 비교, 클릭수, 광고비 |
| CTR 급락 Rule | 소재 피로도 또는 노출 위치 변화 가능성 확인 | CTR 기간 비교, 노출수, 클릭수 |
| CVR 급락 Rule | 상세페이지, 가격, 리뷰, 배송 조건 변화 가능성 확인 | CVR 기간 비교, 클릭수, 전환수 |
| 예산 소진 속도 Rule | 월 예산 대비 소진 속도 과다 여부 확인 | 일자, 광고비, 월 예산 |
| 데이터 부족 Gate | 적은 행이나 적은 전환으로 과한 판단 방지 | 행 수, 클릭수, 전환수, 광고비 |
| 예산 부족 Rule | 성과 부진이 아니라 예산 부족으로 노출 기회가 제한된 상태를 구분 | 광고비, 노출수, 클릭수, 캠페인 예산 |
| 시즌성 영향 Rule | 계절, 행사, 특정 기간 영향으로 인한 변동을 구조적 문제로 오판하지 않기 위함 | 일자, 전주/전월 비교, 시즌 태그, 행사 기간 |

## 9. 광고주 질문 유형 수집

실제 검증 때 광고주가 물어볼 가능성이 높은 질문을 함께 기록합니다.

| 질문 유형 | 예시 | V2 대응 방향 |
| --- | --- | --- |
| 왜 줄여야 하나요 | 이 캠페인은 왜 예산 축소 대상인가요? | 근거 지표와 비교 기간 제시 |
| 왜 늘려야 하나요 | ROAS가 좋으면 얼마나 늘려도 되나요? | 확대 한도와 테스트 방식 제시 |
| 매출이 줄어든 이유 | 광고 문제인가요, 상품 문제인가요? | 광고 요인과 상품 요인을 분리 표현 |
| 이 수치가 좋은 건가요 | CTR 0.8%면 나쁜 건가요? | 업종/매체 기준 필요 |
| 바로 꺼야 하나요 | 전환이 없으면 중단해야 하나요? | 중단, 축소, 관찰을 단계화 |
| 보고서에 넣어도 되나요 | 광고주에게 어떻게 설명하나요? | 광고주 설명 문구 품질 강화 |

## 10. 광고주 설명 문구 품질 평가 기준

| 평가 항목 | 기준 |
| --- | --- |
| 쉬운 표현 | 초보 광고주도 이해 가능한가 |
| 비단정성 | 원인을 가능성/의심/검토 필요 수준으로 표현했는가 |
| 근거 명확성 | 어떤 수치 때문에 판단했는지 보이는가 |
| 행동 연결성 | 그래서 무엇을 해야 하는지 바로 보이는가 |
| 광고주 대응성 | 보고서나 메시지에 그대로 활용 가능한가 |
| 위험 관리 | 추적 오류, 데이터 부족, 상품 요인을 배제하지 않는가 |

## 11. Rule Engine V2 설계 초안

실제 광고주 데이터 검증 후 확정해야 하며, 현재는 초안입니다.

### 11-1. V2 목표

Rule Engine V2는 단순 이상징후 탐지가 아니라 운영자가 오늘 할 일을 판단할 수 있게 해야 합니다.

핵심 출력:

- 우선순위
- 문제
- 원인 가능성
- 추천 행동
- 예상 효과
- 광고주 설명 문구
- 판단 신뢰도
- 데이터 부족 여부

### 11-2. V2 구조

```text
표준화 데이터
↓
데이터 충분성 Gate
↓
매체/캠페인/상품/키워드 컨텍스트 분류
↓
Rule 평가
↓
중복 이슈 병합
↓
우선순위 점수화
↓
광고주 설명 문구 생성
↓
운영자 피드백 기록
```

### 11-3. V2 핵심 개선

| 개선 영역 | 내용 |
| --- | --- |
| 데이터 충분성 | 행 수, 클릭수, 전환수, 광고비가 부족하면 판단 불가 처리 |
| 우선순위 점수 | 심각도뿐 아니라 광고비 규모, 매출 영향, 반복 발생 여부 반영 |
| Rule 신뢰도 | 높음, 보통, 낮음으로 구분 |
| 매체별 기준 | 네이버, G마켓, 옥션, 11번가 기준 분리 |
| 캠페인 유형 | 브랜드, 일반 검색, 쇼핑, 리타겟팅, 신규 확장 구분 |
| 광고주 상태 분류 | 성과 우수, 성과 부진, 전환 문제, 예산 부족, 시즌성 영향을 먼저 구분 |
| 확대 추천 제한 | ROAS만 높다고 확대하지 않고 전환수와 광고비 최소 기준 적용 |
| 설명 문구 품질 | 단정 금지, 데이터 부족 고지, 광고주용 쉬운 표현 |
| 피드백 루프 | 운영자가 유용함/보통/불필요를 기록해 Rule 개선 |

### 11-4. V2에서 당장 넣으면 좋은 Rule

1. 고비용 무전환 Rule
2. CTR 급락 Rule
3. CVR 급락 Rule
4. CPC 급등 Rule
5. 고ROAS 확대 후보 Rule
6. 고ROAS 저볼륨 확대 보류 Rule
7. 데이터 부족 판단 불가 Rule
8. 브랜드 캠페인 별도 기준 Rule
9. 예산 소진 속도 점검 Rule
10. 광고주 보고 필요 Rule
11. 예산 부족 상태 구분 Rule
12. 시즌성 영향 구분 Rule

## 12. 최종 판단

현재는 실제 광고주 데이터 3~5개가 없으므로 오늘 해야 할 일 엔진의 실무 유용성 검증은 완료할 수 없습니다.

다만 검증표와 Rule Engine V2 설계 초안은 준비했습니다.

다음 단계는 새 기능 개발이 아니라 실제 파일 업로드 후 이 문서의 표를 채우는 것입니다.
