# 테스트 데이터 기준

이 프로젝트에서 “분석할 만한 광고 데이터”는 최소한 광고비가 어디에 쓰였고, 그 결과로 클릭/전환/매출이 얼마나 발생했는지 알 수 있는 데이터입니다.

## 최소 필수 컬럼

아래 컬럼만 있으면 기본 분석이 가능합니다.

- 날짜
- 매체
- 캠페인
- 광고그룹
- 노출
- 클릭
- 광고비

이 경우 가능한 분석:

- 노출 대비 클릭률
- 클릭당 비용
- 캠페인별 광고비 비중
- 클릭률 저조 구간 탐지

## 강력 추천 컬럼

아래 컬럼까지 있으면 실제 운영 판단이 가능합니다.

- 전환
- 매출
- 주문수
- 키워드
- 상품명
- 계정명 또는 광고주명

이 경우 가능한 분석:

- 전환율
- 전환당 비용
- 광고수익률
- 광고비 소진 대비 전환 없음 탐지
- 예산 확대 후보 탐지
- 제외 또는 입찰 하향 후보 탐지

## 좋은 테스트 데이터의 조건

테스트 데이터는 너무 깨끗하면 분석 결과가 재미없습니다. 일부러 아래 상황이 섞여 있어야 합니다.

- 잘 되는 캠페인: 전환과 매출이 있고 광고수익률이 높은 행
- 나쁜 캠페인: 광고비는 쓰는데 전환이 없는 행
- 의심 캠페인: 노출은 많은데 클릭률이 낮은 행
- 성장 캠페인: 최근 기간에 클릭/전환/매출이 늘어난 행
- 비교 가능한 기간: 최소 7일 이상, 가능하면 14일 이상

## 행 단위 기준

추천하는 1행의 의미는 아래 중 하나입니다.

- 하루 + 캠페인 1개
- 하루 + 광고그룹 1개
- 하루 + 키워드 1개
- 하루 + 상품 1개

가장 좋은 MVP 입력은 “하루 + 캠페인 또는 광고그룹” 단위입니다. 너무 세밀한 키워드 단위는 나중에 추가해도 됩니다.

## 숫자 입력 기준

숫자는 아래처럼 넣는 것이 가장 안전합니다.

- 광고비: `420000`
- 노출: `13000`
- 클릭: `990`
- 전환: `57`
- 매출: `6840000`

쉼표나 원 표시가 있어도 일부 처리되지만, 처음에는 숫자만 넣는 것이 좋습니다.

## 표시 형식 기준

보고서와 엑셀 결과에서는 아래 형식으로 표시합니다.

- 일반 숫자: `1,603,400`
- 금액: `26,470,000원`
- 비율: `1.41%`
- 광고수익률: `415.00%`

엑셀 원본 값은 계산 가능한 숫자로 유지하고, 표시 형식만 적용합니다.

엑셀 파일에서는 금액도 계산하기 쉽도록 숫자형으로 유지하고 `#,##0` 쉼표 서식을 적용합니다. Markdown 보고서와 업로드 화면에서는 `원`을 붙여 표시합니다.

## 지금 만든 테스트 파일

테스트용 원본 파일:

```text
data/raw/test_ads_full_visible.csv
```

이 파일에는 일부러 아래 패턴을 넣었습니다.

- 브랜드 검색: 성과가 좋은 캠페인
- 쇼핑 검색: 예산 확대 후보
- 경쟁사 키워드: 비용은 쓰지만 전환이 없는 위험 구간
- 디스플레이 신규유입: 노출은 많지만 클릭률이 낮은 구간
- 리타겟팅: 안정적인 전환 캠페인
- 신규 확장 테스트: 검토 또는 정리 후보
