# 지마켓/옥션 광고센터 리포트 학습 메모

작성일: 2026년 6월 8일

## 1. 데이터 출처

이번에 사용자가 제공한 파일은 지마켓/옥션 광고센터에서 확인 가능한 광고 리포트 데이터입니다.

기간 기준:

- 최근 90일 기준
- 파일명 기준 기간: 2026-03-10 ~ 2026-06-07

광고주 또는 테스트 대상:

- 비식별 샘플 광고주

## 2. 전달받은 파일 목록

아래 파일명은 실제 원본을 공개하지 않기 위해 비식별 예시로 기록합니다.

```text
sample_report_AdId_YYYYMMDDYYYYMMDD.xlsx
sample_report_daily_YYYYMMDDYYYYMMDD.xlsx
sample_report_group_YYYYMMDDYYYYMMDD.xlsx
sample_report_monthly_YYYYMMDDYYYYMMDD.xlsx
```

## 3. 파일명 기준 추정 리포트 단위

실제 컬럼 확인 전이므로 아래 내용은 파일명 기반 추정입니다.

| 파일 유형 | 추정 의미 | AIMAOS 연결 후보 |
| --- | --- | --- |
| AdId | 광고 ID 또는 소재 ID 단위 리포트 | 광고/소재 단위 성과 분석 |
| daily | 일자별 리포트 | 일자별 성과, 기간 비교 |
| group | 광고그룹 단위 리포트 | 광고그룹별 성과 분석 |
| monthly | 월별 리포트 | 월별 성과, 월간 보고 |

## 4. AIMAOS에서 중요하게 볼 지점

지마켓/옥션 광고센터 리포트는 기존 광고 데이터 분석 MVP에 다음 방향으로 연결합니다.

- Excel 업로드 대상 파일로 처리
- 컬럼 자동 매핑 대상으로 추가 검토
- 일자별, 월별, 광고그룹별 리포트는 기존 기간 분석 구조와 연결
- AdId 리포트는 향후 광고 또는 소재 단위 분석 확장 후보
- 지마켓/옥션은 채널 통합 관제의 오픈마켓 영역에 포함

## 5. 가장 중요한 리스크

사용자 설명에 따르면 새 지마켓/옥션 광고센터가 나올 예정입니다.

따라서 현재 리포트 구조가 장기간 유지된다고 가정하면 안 됩니다.

설계 원칙:

- 특정 컬럼명에만 의존하지 않습니다.
- 리포트 버전을 구분할 수 있게 합니다.
- 기존 광고센터 리포트와 신규 광고센터 리포트를 별도 어댑터로 분리합니다.
- 컬럼명이 바뀌어도 표준 스키마로 매핑할 수 있게 합니다.
- 실제 로그인 자동화나 크롤링은 현재 단계에서 구현하지 않습니다.
- 우선은 수동 다운로드 Excel 업로드 기반으로 안정화합니다.

## 6. 향후 파서 설계 방향

추천 구조:

```text
aimaos/parsers/marketplace/
  gmarket_auction_legacy.py
  gmarket_auction_next.py
```

다만 현재 단계에서는 실제 파서 구현보다 샘플 파일의 컬럼 구조 확인이 먼저입니다.

## 7. 표준 스키마 연결 후보

실제 컬럼 확인 후 아래 표준 필드로 매핑합니다.

| 표준 필드 | 의미 |
| --- | --- |
| date | 날짜 |
| platform | 지마켓 또는 옥션 |
| account_name | 계정명 또는 광고주명 |
| campaign_name | 캠페인명 |
| ad_group_name | 광고그룹명 |
| ad_id | 광고 ID |
| product_name | 상품명 |
| impressions | 노출수 |
| clicks | 클릭수 |
| cost | 광고비 |
| conversions | 전환수 |
| revenue | 매출 |
| orders | 주문수 |
| ctr | 클릭률 |
| cpc | 클릭당 비용 |
| cvr | 전환율 |
| cpa | 전환당 비용 |
| roas | 광고수익률 |

## 8. 현재 처리 상태

현재 작업 환경에서는 사용자의 비공개 원본 저장소에 직접 접근할 수 없습니다.

따라서 실제 시트명, 컬럼명, 샘플 행, 데이터 타입은 아직 확인하지 못했습니다.

다음 단계에서 파일을 아래 워크스페이스 경로로 복사하면 실제 구조 분석을 진행합니다.

```text
data/raw/gmarket_auction/sample_advertiser/
```

## 9. 다음 확인 항목

파일 접근이 가능해지면 아래 항목을 확인합니다.

- 각 파일의 시트명
- 헤더 시작 행
- 컬럼명
- 행 수
- 날짜 컬럼 형식
- 금액 컬럼 형식
- 퍼센트 컬럼 형식
- 지마켓/옥션 구분 컬럼 존재 여부
- 광고 ID, 상품 ID, 광고그룹 ID 존재 여부
- 기존 AIMAOS 컬럼 매핑과 충돌 여부
- 일자별, 월별, 광고그룹별, 광고 ID별 리포트 간 중복 또는 결합 기준

## 10. 최종 판단

이 데이터는 AIMAOS에서 오픈마켓 광고 분석을 확장하기 위한 중요한 샘플입니다.

단, 새 지마켓/옥션 광고센터가 예정되어 있으므로 현재 파일 구조에 강하게 고정된 구현은 위험합니다.

우선 수동 업로드 기반으로 리포트 구조를 학습하고, 이후 `legacy`와 `next` 구조를 나누어 대응하는 방향이 안전합니다.
