# Streamlit 앱 동기화 계획

## 목적

로컬 최신 `aimaos/app/streamlit_app.py`와 GitHub 원격 배포본의 차이를 정리하고, Streamlit Cloud 데모 안정화를 위해 필요한 동기화 범위를 기록한다.

## 현재 상태

- 로컬 최신 파일: 3,659 lines
- 원격 배포 파일: 3,510 lines
- 차이 규모: 약 149 lines, unified diff 약 414 lines
- 원격 온라인 데모: 현재 안전 데모 데이터 파일 추가로 빈 화면 문제는 해결
- 남은 리스크: 원격 앱 코드에는 로컬 최신 데모 모드와 데이터 상태 개선 일부가 아직 반영되지 않음

## 주요 차이 영역

1. 라이트/다크 모드 가독성 보강 CSS
2. `samples/demo_data/campaignpulse_demo_ads.csv` 기준 데모 데이터 모드
3. 업로드 데이터가 데모 데이터보다 우선되는 세션 스냅샷 구조
4. 데이터 상태 센터에서 데모 데이터 상태를 별도 처리하는 로직
5. 사이드바 `데모 데이터 보기` 토글
6. 광고 성과 분석 화면에서 데모 데이터 안내 문구 표시

## 원격에 아직 남은 표시 차이

원격 구버전 앱은 11번가와 쿠팡을 코드상 `연결 필요`로 고정한다. 그래서 실제 데모 데이터가 보여도 전체 상태가 `주의`로 남을 수 있다.

이 문제는 데이터 파일 추가만으로는 완전히 해결할 수 없고, `streamlit_app.py` 코드 동기화가 필요하다.

## 동기화 권장 방식

1. 로컬 최신 파일에 실제 광고주 데이터나 API Key가 없는지 재확인한다.
2. 로컬 최신 파일을 GitHub 원격의 `aimaos/app/streamlit_app.py`에 전체 교체 방식으로 반영한다.
3. Streamlit Cloud를 재배포한다.
4. 아래 화면을 다시 확인한다.
   - 종합 대시보드
   - 광고 성과 분석
   - 우선 처리 이슈
   - 운영 현황
   - 쇼핑몰/매체 분석
   - 보고서 센터
   - 채널 관리
   - 설정

## 현재 도구 제약

현재 로컬 환경에는 `git`과 `gh`가 설치되어 있지 않다. GitHub 커넥터는 파일 전체 교체는 가능하지만, 159KB 수준의 앱 파일 전체를 직접 전달해야 하므로 현재 세션에서는 무리하지 않았다.

따라서 이번 단계에서는 안전 데모 데이터와 보고서 파일을 원격에 추가해 온라인 빈 화면 문제를 해결했고, 앱 코드 전체 동기화는 별도 안정화 과제로 남긴다.

## 검증 기준

동기화 성공 후에는 다음을 통과해야 한다.

- 온라인 첫 화면에서 데모 데이터 안내 문구 표시
- 사이드바에서 `데모 데이터 보기` 표시
- 데모 데이터 사용 시 데이터 상태가 `데모` 또는 동등한 안내 상태로 표시
- 실제 파일 업로드 시 업로드 데이터가 데모 데이터보다 우선
- `python -m compileall aimaos tests` 성공
- Streamlit AppTest 예외 0건
