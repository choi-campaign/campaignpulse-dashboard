# 실제 광고 파일 검증 요약

작성일: 2026년 6월 8일

## 1. 검증 목적

실제 네이버/G마켓/옥션/11번가 광고 파일을 `data/raw` 폴더에 넣었을 때 AIMAOS가 정상 로드, 컬럼 매핑, 표준화, 오늘 해야 할 일 이슈 생성을 수행할 수 있는지 점검하기 위한 단계입니다.

## 2. 현재 data/raw 상태

현재 `data/raw` 폴더에는 실제 네이버/G마켓/옥션/11번가 원본 광고 리포트 파일이 아직 없습니다.

현재 확인된 파일:

```text
data/raw/sample_ads.csv
data/raw/test_ads_full_visible.csv
data/raw/gmarket_auction/nutirone/README.md
```

따라서 이번 검증은 실제 매체 파일이 아닌 기존 샘플 CSV 2개를 기준으로 수행했습니다.

## 3. 추가한 검증 모듈

추가 파일:

```text
aimaos/validators/raw_media_audit.py
```

역할:

- `data/raw` 하위 CSV/XLSX/XLS 파일 자동 탐색
- 파일 로드 가능 여부 확인
- 현재 `column_mapper.py` 기준 컬럼 매핑 성공/실패 확인
- 필수 지표 표준화 상태 확인
- 오늘 해야 할 일 이슈 생성 수 확인
- 파일별 진단 Markdown 리포트 생성

## 4. 생성된 진단 리포트

```text
data/reports/raw_media_audit/raw_media_audit_report.md
```

## 8. 오늘 해야 할 일 이슈 검토

샘플 기준 생성된 이슈는 다음 유형입니다.

- 광고비는 사용됐지만 전환이 없습니다
- 클릭률이 낮아 광고 반응이 약합니다
- 성과가 좋아 예산 확대를 검토할 수 있습니다

실무적으로 과한 단정으로 보이는 문구는 발견되지 않았습니다.

다만 캠페인 단위 이상 탐지 결과에는 매체 정보가 포함되어 있지 않아, 기존에는 `매체 미분류`로 표시될 수 있었습니다. 이번 검토에서 매체 정보가 없는 경우 `판단 불가`로 표시되도록 수정했습니다.

## 9. 수정한 룰

수정 파일:

```text
aimaos/recommenders/action_engine.py
```

수정 내용:

- 액션 이슈에 매체 정보가 없거나 `매체 미분류`인 경우 `판단 불가`로 표시
- 매체를 임의 추정하지 않음

## 10. 추가로 필요한 컬럼

실제 매체 파일을 기준으로 추가 확인이 필요한 컬럼은 다음입니다.

- 장바구니 수
- 상품 ID
- 광고 ID
- 광고 유형
- 노출 상품명
- 주문수와 전환수의 구분
- 매출 기준
- 직접/간접 전환 구분
- 지마켓/옥션 구분 컬럼
- 광고센터 리포트 버전

## 11. 다음 개선 우선순위

1. 실제 네이버/G마켓/옥션/11번가 파일을 `data/raw` 하위 폴더에 복사
2. `raw_media_audit.py`로 실제 파일 매핑 진단 실행
3. 미매핑 컬럼을 `COLUMN_ALIASES`에 추가
4. 장바구니 수를 표준 스키마에 포함할지 검토
5. 매체별 legacy/next 리포트 구조 분리
6. 캠페인 단위 이상 탐지 결과에 매체 정보를 유지할 수 있는 보조 구조 검토

## 12. 실행 명령

현재 진단 리포트 생성 방식:

```powershell
@'
from pathlib import Path
from aimaos.validators.raw_media_audit import audit_raw_media_files, write_audit_report
audits = audit_raw_media_files(Path('data/raw'))
write_audit_report(audits, Path('data/reports/raw_media_audit/raw_media_audit_report.md'))
'@ | .\.venv\Scripts\python.exe -
```

## 13. 최종 판단

현재 샘플 파일 기준으로는 로드, 매핑, 표준화, 오늘 해야 할 일 엔진이 정상 동작합니다.

하지만 실제 네이버/G마켓/옥션/11번가 파일이 아직 `data/raw`에 없기 때문에, 매체별 실제 컬럼 구조와 실무 적합성은 아직 최종 검증하지 못했습니다.

실제 파일이 들어오면 이 검증 모듈로 바로 매핑 성공/실패와 Rule Engine 이슈 품질을 확인할 수 있습니다.
