## 2. 네이버 검색광고

확인 완료:

- Customer ID, Access License, Secret Key 인증
- 캠페인 조회 HTTP 200, 1건
- 광고그룹 조회 HTTP 200, 2건
- 광고그룹 ID 기준 키워드 조회 성공, 0건
- stats API 조회 성공
- 성공 조합: 캠페인 반복 `ids`와 rich fields
- `response.data` 빈 배열 처리
- `naver_searchad_standard.csv` 생성
- 기존 분석 파이프라인 연결
- 오늘 해야 할 일 생성
- Markdown, TXT, Excel 보고서 생성

주의:

- 검증한 날짜의 `response.data`가 비어 있어 실제 양수 성과 행은 아직 확인하지 못했다.
- 전환·매출·ROAS는 계정의 전환 추적 설정과 stats 응답 필드에 따라 조건부다.
- 대행사 계정에서 접근 가능한 여러 광고주의 전체 목록 조회는 Customer ID 운영 방식에 맞춘 추가 검증이 필요하다.

네이버 API는 연결 실패가 아니라 기술 검증 성공으로 분류한다. 운영 단계의 남은 과제는 스케줄링, 재시도, 암호화된 비밀값 관리, 수집 로그와 알림이다.

## 3. G마켓·옥션

확인 완료:

- 전용 수집 브라우저 실행
- 사용자가 직접 ESM 로그인
- 광고센터 진입
- 새 탭·새 창 처리
- Excel, XLS, CSV 다운로드 감지 구조
- 미완료 파일 제외
- `demo_` 체험 파일 제외
- 수집 시작 이전의 과거 파일 제외
- `collection_log` 성공·실패 기록 구조

아직 미검증:

- 최근 30~90일 광고 집행 이력이 있는 계정의 리포트 메뉴 진입
- 기간 선택과 조회
- 실제 Excel 또는 CSV 생성
- AIMAOS 다운로드 감지
- `raw_media_audit.py` 진단
- 기존 분석 파이프라인 연결
- 오늘 해야 할 일과 보고서 생성
- `collection_log` success 기록

광고센터 진입은 수집 성공이 아니다. 실제 리포트 파일이 생성되고 AIMAOS가 감지한 경우에만 성공으로 기록한다.

## 4. 11번가

실제 계정과 원본 파일이 없어 검증 보류다. 로그인·광고센터·리포트 다운로드 경로가 확보되기 전에는 성공 가능성을 확정하지 않는다.

## 5. 보안 원칙

- 실제 광고주 비밀번호를 저장하지 않는다.
- 캡차와 2차 인증은 사용자가 직접 처리한다.
- `.env`, API Key, Secret Key, 실제 광고주 원본 파일은 GitHub에 올리지 않는다.
- GitHub와 Streamlit Cloud에는 가짜 데모 데이터만 포함한다.
- API 비밀값은 운영 전 암호화 저장과 접근 권한 분리가 필요하다.

