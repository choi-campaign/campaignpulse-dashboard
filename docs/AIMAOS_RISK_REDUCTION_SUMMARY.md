# AIMAOS Risk Reduction Summary

## 기준 질문

광고주가 더 편해지는가?

이번 작업의 판단 기준은 UI 완성도가 아니라 데이터 수집 부담 감소와 운영 안정성이다.

## 현재 구현한 리스크 감소 장치

### 1. Data Status Center 우선 진입

기존 흐름은 사용자가 바로 오늘 해야 할 일과 보고서를 볼 수 있었다.

이제 첫 화면에서 데이터 최신성을 먼저 확인한다.

- 마지막 수집 시각
- 마지막 분석 시각
- 마지막 성공 시각
- 마지막 실패 시각
- 실패 원인
- 매체별 연결 상태

이를 통해 오래된 데이터로 잘못된 운영 판단을 하는 위험을 줄인다.

### 2. 네이버 자동 수집 가능성 검증

네이버 검색광고 API는 기술 검증 성공 상태다.

- API 호출 성공
- 캠페인/광고그룹/stats 조회 성공
- 표준 CSV 생성 성공
- 기존 분석 파이프라인 연결 성공
- 보고서 생성 성공

네이버는 파일 다운로드/업로드 제거 가능성이 높다.

### 3. G마켓/옥션 반자동 수집 방향 확정

현재 검증된 것:

- 전용 수집 브라우저 실행
- ESM 로그인
- 광고센터 진입
- 새 창 처리

보류된 것:

- 최근 광고 데이터가 있는 계정에서 리포트 다운로드 검증
- 다운로드 파일 자동 감지
- 분석 파이프라인 자동 연결

### 4. 수집 로그 구조 추가

`collection_log` 구조를 추가해 수집 성공/실패 원인을 축적할 수 있게 했다.

이 로그는 고객지원 비용 감소와 95% 이상 수집 성공률 관리에 필요하다.

### 5. 보관 정책과 정리 명령 추가

원본/가공/보고서/로그/스크린샷 보관 기간을 구분했다.

저장 비용이 커지는 리스크를 미리 통제한다.

## 아직 남은 가장 큰 리스크

### 1. G마켓/옥션 다운로드 검증

광고센터 진입은 성공했지만, 최근 광고 데이터가 있는 계정에서 다운로드까지 검증해야 한다.

이 단계가 통과되어야 광고주 파일 다운로드/업로드 제거가 현실화된다.

### 2. 로그인 세션 유지

광고주가 매일 로그인해야 하면 사용성이 크게 떨어진다.

세션 유지 가능성은 별도 정책/보안 검토가 필요하다.

### 3. 매체 구조 변경

G마켓/옥션 광고센터 개편 예정이 있어 legacy 구조에만 의존하면 위험하다.

legacy와 next profile을 분리 유지해야 한다.

### 4. 쿠팡 보안 정책

쿠팡 광고센터는 자동화에 민감할 수 있어 위험 매체로 분류한다.

## 판매를 막는 현재 병목

광고주가 데이터를 직접 내려받아 업로드해야 하는 순간 판매 난이도가 급격히 올라간다.

따라서 다음 판매 가능 MVP 조건은 다음과 같다.

1. 네이버는 API 자동 수집
2. G마켓/옥션은 주 1회 이상 반자동 수집
3. 수집 실패 시 사용자가 원인을 이해 가능
4. 실패해도 기존 데이터로 계속 보기 가능

## 다음 우선순위

1. 최근 광고 집행 G마켓/옥션 계정으로 리포트 다운로드 검증
2. 다운로드 파일 감지 후 raw_media_audit 연결
3. 분석 파이프라인 자동 실행
4. collection_log에 실제 수집 결과 기록
5. Data Status Center에서 30일 성공률 계산

## 사업성 판단

조건부 가능.

네이버는 기술 검증 성공으로 자동 수집 가능성이 높다.

G마켓/옥션은 광고센터 진입까지 성공했으나 다운로드 검증이 남아 있다. 이 검증이 통과되면 실제 판매 가능한 MVP에 가까워진다.
