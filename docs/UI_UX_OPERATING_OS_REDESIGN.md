# AIMAOS UI/UX 운영 OS 재설계 기록

작성일: 2026년 6월 7일

## 1. 작업 목적

이번 작업은 신규 분석 기능 추가가 아니라 기존 광고 데이터 분석 MVP를 보호하면서 화면 흐름을 사용자 행동 중심으로 재정리한 작업입니다.

핵심 목표는 사용자가 아래 질문에 빠르게 답할 수 있도록 만드는 것입니다.

- 광고가 잘 되고 있는가
- 광고비가 낭비되고 있는가
- 돈을 벌고 있는가
- 무엇을 먼저 점검해야 하는가
- 광고주에게 무엇을 보고해야 하는가

## 2. 보호한 MVP 기능

아래 기능은 계산 로직을 변경하지 않았습니다.

- CSV 업로드
- Excel 업로드
- 컬럼 자동 매핑
- 표준화
- 광고 성과 분석
- 운영 점검 포인트
- 기간 분석
- 비교 분석
- 보고서 생성

## 3. 메뉴 구조 변경

기존 기능 중심 메뉴를 운영 흐름 중심 메뉴로 재구성했습니다.

| 변경 후 메뉴 | 역할 |
| --- | --- |
| 대시보드 | 전체 상태, AI 운영 코치, 핵심 KPI, 협회 자산화 현황을 먼저 보여줍니다. |
| 광고 분석 | 기존 광고 데이터 분석 MVP입니다. |
| 쇼핑몰 분석 | Commerce Intelligence 준비 화면입니다. |
| 보고서 센터 | 생성된 보고서 파일을 한곳에서 확인하고 다운로드합니다. |
| 채널 관리 | 채널 통합 관제와 채널 DB 관리 화면을 묶었습니다. |
| 설정 | MVP 보호 기능과 시스템 상태를 확인합니다. |
| Beta 영역 | 검색 인텔리전스와 AI 가시성을 별도 준비중 영역으로 이동했습니다. |

## 4. 대시보드 변경

홈 대시보드는 데이터 표 중심이 아니라 행동 중심으로 구성했습니다.

표시 순서:

1. 광고 건강도
2. 전체 상태
3. AI 운영 코치
4. 핵심 KPI
5. 매체별 성과
6. 협회 자산화 현황
7. 전문가용 빠른 진단

광고 건강도는 화면 판단을 돕기 위한 UI 요약값이며, 기존 분석 결과값을 변경하지 않습니다.

## 5. AI 운영 코치

기존 운영 점검 포인트를 더 중요한 위치로 올렸습니다.

분류 기준:

- 위험
- 기회
- 관찰

각 영역은 광고 운영자가 바로 행동할 수 있도록 핵심 대상과 판단 근거를 먼저 보여줍니다.

## 6. 기본 모드와 전문가 모드

광고 분석 화면에 보기 모드를 추가했습니다.

기본 모드:

- 광고비
- 매출
- 주문수
- 광고수익률

전문가 모드:

- 노출수
- 클릭수
- 클릭률
- 클릭당 비용
- 전환수
- 전환율
- 전환당 비용

기본 모드는 대표와 판매자에게 맞추고, 전문가 모드는 광고 실무자와 대행사 담당자에게 맞췄습니다.

## 7. 카드 UI 개선

핵심 지표 카드는 기존보다 크게 보이도록 조정했습니다.

카드에는 다음 정보를 포함합니다.

- 현재 수치
- 이전 기간 대비 증감률
- 지표 설명

## 8. 테이블 UX 개선

모든 데이터를 한 번에 노출하지 않고 보기 모드에 따라 표시 컬럼을 줄였습니다.

기본 모드에서는 핵심 KPI만 보여주고, 전문가 모드에서는 세부 지표를 추가로 보여줍니다.

## 9. 채널 통합 관제 개선

채널 구조는 유지하면서 화면 순서를 조정했습니다.

표시 순서:

1. 전체 상태
2. AI 채널 운영 코치
3. 채널 영역
4. 통합 KPI
5. 협회 자산화 현황
6. 채널 벤치마크
7. 채널별 샘플 성과

채널은 기존 원칙대로 DB 기반으로 관리합니다.

## 10. 협회 자산화 영역

AIMAOS가 단순 분석기가 아니라 협회 운영 자산을 축적하는 시스템임을 보여주기 위해 자산화 지표를 추가했습니다.

표시 항목:

- 누적 업종 수
- 누적 광고 데이터 수
- 벤치마크 수
- 운영 사례 수
- 관리 채널 수

현재 실데이터 벤치마크는 연결 전이므로 과장하지 않고 준비 상태로 표시합니다.

## 11. 검증 결과

확인한 항목:

- Streamlit 앱 화면 렌더링 정상
- 대시보드 표시 정상
- 광고 분석 메뉴 표시 정상
- 샘플 광고 데이터 분석 실행 정상
- 보고서 다운로드 영역 표시 정상
- 보고서 센터 표시 정상
- 채널 관리 표시 정상
- 설정 화면 표시 정상
- 기존 샘플 분석 파이프라인 정상

pytest는 현재 가상환경에 설치되어 있지 않아 실행하지 못했습니다. 대신 샘플 파일로 실제 분석 파이프라인을 실행해 Markdown, TXT, Excel 보고서 생성까지 확인했습니다.

## 12. 최종 판단

이번 작업은 분석 엔진 변경 없이 AIMAOS의 사용 흐름을 운영 OS에 가깝게 정리한 단계입니다.

이제 첫 화면에서 전체 상태를 보고, AI 운영 코치에서 무엇을 먼저 점검할지 확인하고, 필요할 때 전문가 모드로 들어가 원인을 찾는 구조가 되었습니다.
