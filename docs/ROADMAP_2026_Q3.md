# AIMAOS 2026년 3분기 로드맵

## 1. 문서 목적

이 문서는 AIMAOS MVP 1인 광고 데이터 분석기를 유지하면서, 2026년 3분기 이후 확장할 Phase 2 구조를 정의하기 위한 로드맵입니다.

현재 광고 데이터 분석 MVP는 핵심 자산으로 유지합니다. 이번 로드맵은 신규 수집 기능 구현이 아니라 향후 확장을 위한 메뉴, 화면, 데이터 구조 설계에 집중합니다.

## 2. 현재 유지 원칙

- 기존 광고 데이터 분석기 구조를 변경하거나 제거하지 않습니다.
- 실제 광고 데이터 분석 로직을 변경하지 않습니다.
- 현재 보고서 생성 기능을 최우선 자산으로 유지합니다.
- Phase 2 기능은 샘플 미리보기 상태로만 제공합니다.
- 실제 크롤러 구현은 하지 않습니다.
- 실제 로그인 자동화 구현은 하지 않습니다.
- 실제 데이터 수집 실행은 하지 않습니다.

## 3. Phase 2 상위 모듈

Phase 2에서는 아래 3개 Intelligence 모듈을 상위 구조로 추가합니다.

| 모듈 | 목적 | 현재 상태 |
| --- | --- | --- |
| Commerce Intelligence | 커머스 상품, 가격, 리뷰, 경쟁상품 흐름을 자산화 | 샘플 미리보기 |
| Search Intelligence | 검색 키워드, 검색량, 경쟁사 노출 흐름을 자산화 | 샘플 미리보기 |
| AI Visibility Intelligence | 생성형 AI 답변 내 브랜드 언급과 추천 여부를 자산화 | 샘플 미리보기 |

## 4. Streamlit 메뉴 구조

현재 메뉴 구조는 아래와 같습니다.

```text
AIMAOS
  광고 데이터 분석기
  채널 통합 관제
  채널 관리자
  커머스 인텔리전스
  검색 인텔리전스
  AI 가시성
```

`광고 데이터 분석기`는 기존 MVP 화면입니다.

`채널 통합 관제`와 `채널 관리자`는 Channel Intelligence Layer의 초기 구조입니다. 채널 목록은 SQLite DB에서 관리하며, 실제 수집이나 자동화 기능은 포함하지 않습니다.

커머스 인텔리전스, 검색 인텔리전스, AI 가시성 메뉴는 Phase 2 확장 준비 화면입니다. 현재는 테스트 샘플 데이터 미리보기만 제공하며, 실제 수집이나 자동화 기능은 포함하지 않습니다.

## 5. Commerce Intelligence 로드맵

### 대상

- 스마트스토어
- 쿠팡
- G마켓
- 옥션
- 11번가

### 향후 저장 예정 데이터

- 상품명
- 가격
- 리뷰
- 평점
- 판매량
- 경쟁상품

### 3분기 설계 목표

1. 상품 단위 데이터 표준 스키마 정의
2. 플랫폼별 상품 식별 기준 설계
3. 가격 변동 이력 저장 구조 설계
4. 리뷰와 평점 변화 추적 구조 설계
5. 광고 성과와 상품 경쟁력 데이터를 연결할 수 있는 키 설계

### MVP와의 연결 방향

광고 데이터 분석기에서 성과가 낮은 캠페인이 발견되었을 때, 단순히 광고 설정만 보는 것이 아니라 상품 가격, 리뷰, 평점, 경쟁상품 상태까지 함께 판단할 수 있게 확장합니다.

## 6. Search Intelligence 로드맵

### 대상

- 네이버
- 구글
- 카카오다음
- 네이트

### 향후 저장 예정 데이터

- 키워드
- 검색량
- 경쟁도
- 검색 결과
- 경쟁사 노출

### 3분기 설계 목표

1. 키워드 기준 데이터 스키마 정의
2. 검색량과 경쟁도 저장 구조 설계
3. 검색 결과 관측 이력 구조 설계
4. 경쟁사 노출 여부 기록 기준 설계
5. 광고 성과와 검색 수요 데이터를 연결할 기준 설계

### MVP와의 연결 방향

광고 데이터 분석기에서 클릭률이 낮거나 전환이 약한 키워드가 발견되면, 해당 키워드의 검색 수요와 경쟁 상황을 함께 검토할 수 있게 확장합니다.

## 7. AI Visibility Intelligence 로드맵

### 대상

- ChatGPT
- Claude
- Gemini
- Perplexity

### 향후 저장 예정 데이터

- 브랜드 언급
- 경쟁사 언급
- 답변 출처
- 추천 여부

### 3분기 설계 목표

1. 질문 세트 저장 구조 설계
2. AI 답변 관측 결과 저장 구조 설계
3. 브랜드 언급 여부 기록 기준 설계
4. 경쟁사 언급 여부 기록 기준 설계
5. 답변 출처와 추천 여부 저장 구조 설계

### MVP와의 연결 방향

광고 성과가 좋은 브랜드 또는 상품이 생성형 AI 답변에서도 추천되는지 확인하고, 검색 광고와 AI 검색 노출을 함께 관리하는 방향으로 확장합니다.

## 8. 2026년 3분기 우선순위

### 1순위

현재 광고 데이터 분석 MVP 안정화와 실제 광고주 엑셀 샘플 대응입니다.

이유:

- 현재 수익화 가능성이 가장 높음
- 데이터 자산화의 시작점임
- 협회 표준 보고서로 바로 확장 가능함

### 2순위

Phase 2 데이터 스키마 확정입니다.

이유:

- 수집 구현 전에 저장 기준이 먼저 필요함
- 기준 없이 수집을 시작하면 장기 자산화가 어려움
- 향후 벤치마크와 지식 자산화의 기반이 됨

### 3순위

수동 업로드 기반의 Commerce/Search/AI Visibility 샘플 데이터 입력 구조입니다.

이유:

- 크롤러 없이도 구조 검증 가능
- 법적·정책적 리스크를 낮출 수 있음
- 실제 수집 자동화 전 데이터 모델을 검증할 수 있음

## 9. 현재 구현하지 않는 것

아래 항목은 이번 단계에서 구현하지 않습니다.

- 커머스 플랫폼 크롤러
- 검색엔진 크롤러
- 생성형 AI 자동 질의 수집기
- 광고센터 로그인 자동화
- 커머스 계정 로그인 자동화
- 캡차 우회
- 보안 우회
- 정책 위반 가능성이 있는 자동화
- 기존 광고 분석 엔진 변경

## 10. 다음 개발 기준

Phase 2 기능을 실제로 구현하기 전에 아래 질문을 먼저 검토합니다.

- 이 데이터가 광고 성과 개선에 직접 도움이 되는가?
- 이 데이터가 협회의 장기 자산으로 쌓이는가?
- 이 기능이 운영 시간을 줄이는가?
- 이 기능이 수익화 가능한 서비스로 연결되는가?
- 정책 위반이나 보안 리스크가 없는가?

위 기준을 통과하지 못하면 후순위로 둡니다.
