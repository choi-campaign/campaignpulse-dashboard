# 마켓플레이스 다운로드 성공 판정 방어

작성일: 2026-06-20

## 목적

G마켓·옥션 수집 POC에서 체험용 파일이나 과거 다운로드 파일을 새로운 수집 성공으로 잘못 기록하지 않도록 성공 판정 기준을 고정한다.

## 적용 기준

- 감지 대상은 `.xlsx`, `.xls`, `.csv` 리포트다.
- 확장자 대소문자와 관계없이 리포트 파일을 감지한다.
- 다운로드 중인 `.crdownload`, `.tmp`, `.part` 파일은 완료 파일로 인정하지 않는다.
- 크기가 0바이트인 빈 파일은 완료 파일로 인정하지 않는다.
- 파일명이 `demo_`로 시작하는 체험용 파일은 실제 수집 성공으로 인정하지 않는다.
- 전용 G마켓 수집 실행은 실행 시작 시각 이후 생성된 파일만 인정한다.
- 공통 마켓플레이스 수집에서 `--open-profile`을 사용하면 수동 브라우저 세션 시작 이후 생성된 파일만 인정한다.
- 단순 진단 실행은 기존 실제 파일을 다시 점검할 수 있도록 날짜 제한을 강제하지 않는다.
- 실제 파일 감지 후에도 원본 진단과 기존 분석 파이프라인 연결이 완료되어야 최종 성공으로 판단한다.


## 현재 판정

다운로드 감지 구조와 성공 오판 방어는 준비됐다. 그러나 실제 광고 계정에서 리포트 파일이 생성되고 CampaignPulse가 이를 감지한 증거는 아직 없으므로 G마켓·옥션 자동 수집 성공으로 기록하지 않는다.

다음 성공 기준:

```text
실제 리포트 파일 생성
→ CampaignPulse 감지
→ raw_media_audit 진단
→ 기존 분석 파이프라인 연결
→ 오늘 해야 할 일 생성
→ 보고서 생성
→ collection_log success 기록
```
