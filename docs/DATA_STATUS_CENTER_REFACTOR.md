# CampaignPulse Data Status Center

최종 점검일: 2026-06-19

## 변경 이유

CampaignPulse의 분석 결과는 데이터가 최신일 때만 신뢰할 수 있다. 사용자가 오래된 데이터를 최신 결과로 오해하지 않도록 수집 상태, 마지막 수집 시각, 실패 원인을 분석 결과보다 먼저 확인할 수 있게 구성했다.

## 현재 화면 구조

현재 최상위 첫 메뉴는 `종합 대시보드`다. 별도의 개발자용 관제 화면을 강제로 앞세우지 않고, 종합 대시보드 상단에 데이터 상태를 통합했다.

```text
종합 대시보드
├─ 오늘의 핵심 요약
├─ 데이터 상태 요약
│  ├─ 전체 상태
│  ├─ 마지막 수집
│  ├─ 마지막 분석
│  ├─ 최근 실패 원인
│  └─ 매체별 데이터 상태
├─ 우선 처리 이슈 요약
├─ 성과 흐름 요약
└─ 보고서 상태
```

세부 데이터 최신성 화면 함수는 유지되어 있으며, 종합 대시보드와 채널 관리가 같은 상태 계산 함수를 공유한다.

## 상태 정의

| 상태 | 기준 | 의미 |
|---|---|---|
| 최신 | 마지막 성공 데이터가 24시간 이내 | 정상 이용 |
| 주의 | 24시간 초과 72시간 이내 또는 일부 수집 | 최신화 권장 |
| 오래됨 | 72시간 초과 | 최신 데이터 수집 우선 권장 |
| 연결 안됨 | 데이터 없음, 최근 수집 실패, 계정 미연결 | 연결 또는 실패 원인 확인 |
| 데모 | 안전한 가짜 데이터 사용 | 기능 시연용이며 실제 광고주 데이터 아님 |

## 실제 상태 계산 순서

1. `data/collection_poc`, `data/raw`에서 최근 수집 파일 시각을 확인한다.
2. `data/storage/collection_log.sqlite3`가 있으면 매체별 최신 수집 로그를 읽는다.
3. 파일 시각과 수집 로그 종료 시각 중 더 최근 값을 전체 마지막 수집 시각으로 사용한다.
4. 매체별 상태는 `collection_log`의 가장 최근 시도를 우선 반영한다.
5. 데모 모드에서는 네이버, G마켓, 옥션, 11번가를 `데모`로 명확히 표시한다.
6. 실제 업로드 데이터가 있으면 데모보다 업로드 결과를 우선한다.

## 매체별 처리

| 최근 로그 | 연결 상태 | 데이터 상태 | 표시 내용 |
|---|---|---|---|
| `success` | 연결 완료 | 최신성 시각에 따라 최신/주의/오래됨 | 마지막 성공 시각, 성공률 |
| `partial` | 기존 연결 상태 유지 | 주의 | 일부 데이터 확인 필요 |
| `no_data` | 기존 연결 상태 유지 | 연결 안됨 | 선택 기간 데이터 없음 |
| `failed` | 기존 연결 상태 유지 | 연결 안됨 | 오류 메시지 또는 오류 코드 |
| 로그 없음 | 기존 POC 증거 기준 | 기존 기준 | 미연결 또는 다운로드 미검증 |

G마켓과 옥션은 광고센터 진입만으로 `success`가 되지 않는다. 실제 리포트 파일 생성과 CampaignPulse 감지가 확인되어야 성공 로그를 남긴다.

## 사용자 안내 정책

- 최신: 모든 화면 정상 이용
- 주의: 최신 데이터 수집 권장 배너 표시
- 오래됨: 오래된 데이터 안내와 최신화 권장
- 연결 안됨: 데이터 연결 또는 인증 확인 안내
- 데모: `현재 화면은 기능 시연을 위한 데모 데이터 기준입니다. 실제 광고주 데이터가 아닙니다.` 표시

서비스 전체를 잠그지 않는다. 수집 장애가 있어도 기존 데이터는 확인할 수 있게 하고, 데이터 신뢰도와 실패 원인을 분명히 보여준다.

## 영향 범위

변경된 영역:

- `aimaos/storage/collection_log.py`
- `aimaos/app/streamlit_app.py`의 데이터 상태 계산과 표시
- `tests/test_marketplace_collection.py`

변경하지 않은 영역:

- 광고 성과 계산 로직
- KPI 계산 로직
- Rule Engine
- 데이터 표준화와 컬럼 매핑
- 업로드와 파싱 로직
- 보고서 계산 및 산출물 구조

## 현재 검증 상태

- 네이버: API 수집과 표준 CSV, 분석 파이프라인 연결 기술 검증 완료
- G마켓: 광고센터 진입 완료, 실제 리포트 다운로드 검증 필요
- 옥션: 광고센터 진입 완료, 실제 리포트 다운로드 검증 필요
- 11번가: 실제 계정 기반 수집 검증 보류
- 쿠팡: 인증과 광고센터 구조 검증 보류

## 남은 과제

1. 최근 광고 집행 이력이 있는 G마켓·옥션 계정으로 실제 파일 다운로드를 검증한다.
2. 다운로드 감지 성공과 실패를 `collection_log`에 실제 기록한다.
3. 네이버 운영 수집 스케줄러가 매 실행 결과를 같은 로그에 기록하도록 연결한다.
4. 광고주별 성공률을 분리하려면 상태 조회 함수에 `advertiser_id` 필터를 추가한다.
5. `최신 데이터 수집` 동작을 운영 실행 큐와 연결하기 전 인증 만료와 중복 실행 정책을 확정한다.
