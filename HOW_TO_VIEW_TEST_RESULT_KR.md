# 확장 테스트 결과 보는 법

`run_full_test_report.bat`을 실행하면 화면 자체가 바뀌는 것이 아니라, 아래 파일들이 새로 생성되거나 덮어쓰기 됩니다.

## 1. 먼저 볼 파일

원본 테스트 데이터:

```text
data/raw/test_ads_full_visible.csv
```

이 파일은 “광고센터에서 다운로드한 엑셀/CSV가 이런 모양이면 된다”는 예시입니다.

## 2. 분석 결과 보고서

자동 생성된 보고서:

```text
data/reports/test_ads_full_visible/report.md
```

여기에서 봐야 할 핵심은 아래 3개입니다.

- 핵심 요약
- 우선 실행 권고
- 이상징후

## 3. 엑셀 분석 결과

자동 생성된 엑셀:

```text
data/reports/test_ads_full_visible/analysis.xlsx
```

엑셀 파일을 열어둔 상태에서 다시 실행하면 Windows가 기존 파일을 잠글 수 있습니다. 이 경우 새 파일이 아래처럼 생성됩니다.

```text
data/reports/test_ads_full_visible/analysis_YYYYMMDD_HHMMSS.xlsx
```

`run_full_test_report.bat`은 가장 최근에 생성된 `analysis*.xlsx` 파일을 자동으로 엽니다.

엑셀 안에는 여러 시트가 있습니다.

- `이용가이드`: 초보자를 위한 파일 사용법과 지표 설명
- `표준화데이터`: 컬럼이 표준화된 원본 데이터
- `전체요약`: 전체 성과 요약
- `실행권고`: 우선 실행할 개선안
- `이상징후`: 자동 감지된 위험 또는 기회 항목
- `기간비교`: 이전 기간과 최근 기간 비교
- `매체별성과`: 매체별 성과
- `캠페인별성과`: 캠페인별 성과
- `광고그룹별성과`: 광고그룹별 성과
- `키워드별성과`: 키워드별 성과
- `상품별성과`: 상품별 성과
- `일자별성과`: 일자별 성과
- `주간별성과`: 주간별 성과
- `월별성과`: 월별 성과
- `분기별성과`: 분기별 성과
- `반기별성과`: 반기별 성과
- `연도별성과`: 연도별 성과
- `시즌별성과`: 시즌별 성과

## 4. 왜 그대로처럼 보였는가

기존 샘플 보고서는 아래 위치입니다.

```text
data/reports/report.md
```

확장 테스트 보고서는 아래 위치입니다.

```text
data/reports/test_ads_full_visible/report.md
```

서로 다른 파일이라서 기존 `report.md`만 열어보면 변화가 없어 보일 수 있습니다.

## 5. 보고서에서 확인할 포인트

확장 테스트 보고서에는 아래처럼 분석 결과가 보여야 합니다.

```text
[긴급] 경쟁사 키워드: 입찰가/예산 하향 또는 검색어 제외 검토
[중요] 쇼핑 검색: 일 예산 확대 또는 유사 키워드/상품 확장
[중요] 디스플레이 신규유입: 소재 문구, 상품명, 키워드 매칭 방식 점검
```

이게 보이면 정상입니다.
