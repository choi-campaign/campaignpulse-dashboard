# AIMAOS 전체 설명서

작성일: 2026년 6월 7일

## 1. 한 줄 정의

AIMAOS는 광고 데이터, 판매 채널 데이터, 마케팅 운영 지식을 통합해서 분석하고 보고서화하며, 장기적으로 AI마케팅협회의 운영 자산으로 축적하기 위한 통합 마케팅 운영 관제 시스템입니다.

현재 단계의 핵심은 거대한 플랫폼을 한 번에 만드는 것이 아니라, 실제 현장에서 바로 쓸 수 있는 광고 데이터 분석 MVP를 먼저 안정화하고, 이후 커머스, 검색, AI 노출, 채널 통합 관제로 확장할 수 있는 구조를 준비하는 것입니다.

## 2. 왜 이 프로젝트를 만드는가

광고 운영 현장에서는 매체별 관리자 화면, 엑셀 보고서, 수기 정리, 반복 분석, 담당자 경험에 의존하는 일이 많습니다. AIMAOS는 이 과정을 표준화해서 운영 시간을 줄이고, 데이터와 노하우가 협회 자산으로 남도록 만드는 것을 목표로 합니다.

핵심 판단 기준은 다음과 같습니다.

- 이 기능이 실제 수익을 더 빨리 만들 수 있는가
- 이 기능이 AI마케팅협회의 데이터 자산으로 축적되는가
- 이 기능이 운영 시간을 줄여주는가
- 이 기능이 경쟁사가 쉽게 따라 하기 어려운 구조인가

## 3. 현재 MVP의 핵심 기능

현재 AIMAOS MVP는 광고 데이터 분석기입니다.

사용자는 광고센터에서 내려받은 Excel 또는 CSV 파일을 업로드하고, 시스템은 자동으로 컬럼을 매핑한 뒤 광고 성과를 분석합니다. 분석 결과는 화면에서 확인할 수 있고, Markdown, TXT, Excel 보고서로 저장됩니다.

현재 가능한 작업은 다음과 같습니다.

- CSV 파일 업로드
- Excel 파일 업로드
- 광고 데이터 컬럼 자동 매핑
- 표준 광고 성과 지표 계산
- 전체 성과 요약
- 매체별 성과 비교
- 캠페인별 성과 분석
- 광고그룹별 성과 분석
- 키워드별 성과 분석
- 운영 점검 포인트 탐지
- 실행 권고 생성
- 기간별 성과 시트 생성
- 사용자 지정 기간 분석
- 두 기간 비교 분석
- Markdown 보고서 생성
- TXT 보고서 생성
- Excel 분석 파일 생성

## 4. 현재 화면 구조

Streamlit 앱의 주요 메뉴는 다음과 같습니다.

| 메뉴 | 현재 역할 |
| --- | --- |
| 광고 데이터 분석기 | 기존 MVP 핵심 화면입니다. Excel 또는 CSV 광고 데이터를 업로드하고 분석합니다. |
| 채널 통합 관제 | 광고매체, 오픈마켓, 커머스 플랫폼, 글로벌 채널을 통합해서 보는 확장 관제 화면입니다. 현재는 샘플 데이터 기반입니다. |
| 채널 관리자 | 채널을 DB에서 추가, 수정, 비활성화할 수 있는 관리 화면입니다. |
| 커머스 인텔리전스 | 향후 스마트스토어, 쿠팡, G마켓, 옥션, 11번가 등 커머스 데이터 분석을 위한 준비 화면입니다. |
| 검색 인텔리전스 | 향후 네이버, 구글, 카카오다음, 네이트 검색 데이터 분석을 위한 준비 화면입니다. |
| AI 가시성 | 향후 ChatGPT, Claude, Gemini, Perplexity에서 브랜드 언급과 추천 여부를 점검하기 위한 준비 화면입니다. |

## 5. 광고 데이터 분석 흐름

현재 광고 데이터 분석기는 아래 순서로 동작합니다.

1. 사용자가 CSV 또는 Excel 파일을 업로드합니다.
2. 시스템이 파일을 읽고 컬럼명을 확인합니다.
3. 매체별로 다른 컬럼명을 공통 스키마로 자동 매핑합니다.
4. 숫자, 날짜, 비율 데이터를 정리합니다.
5. 노출수, 클릭수, 광고비, 전환수, 매출, 주문수를 기준으로 핵심 지표를 계산합니다.
6. 캠페인, 광고그룹, 키워드 단위 성과를 집계합니다.
7. 운영 점검 포인트와 실행 권고를 생성합니다.
8. 화면에 분석 결과를 보여줍니다.
9. Markdown, TXT, Excel 보고서를 생성합니다.

## 6. 현재 표준 지표

현재 시스템에서 사용하는 주요 지표는 다음과 같습니다.

| 한글명 | 영문명 | 의미 |
| --- | --- | --- |
| 노출수 | Impressions | 광고가 사용자 화면에 보인 횟수입니다. |
| 클릭수 | Clicks | 사용자가 광고를 클릭한 횟수입니다. |
| 클릭률 | CTR | 노출수 대비 클릭수 비율입니다. |
| 클릭당 비용 | CPC | 클릭 1회를 얻는 데 든 평균 광고비입니다. |
| 광고비 | Cost | 광고 집행에 사용한 비용입니다. |
| 전환수 | Conversions | 구매, 문의, 신청 등 목표 행동이 발생한 횟수입니다. |
| 전환율 | CVR | 클릭수 대비 전환수 비율입니다. |
| 전환당 비용 | CPA | 전환 1건을 만드는 데 든 평균 비용입니다. |
| 매출 | Revenue | 광고를 통해 발생한 매출 또는 전환 가치입니다. |
| 주문수 | Orders | 광고를 통해 발생한 주문 수입니다. |
| 광고수익률 | ROAS | 광고비 대비 매출 비율입니다. AIMAOS에서는 1배를 100.00%, 2배를 200.00%, 10배를 1000.00%로 표시합니다. |

## 7. 표시 형식 개선 사항

초보자도 쉽게 볼 수 있도록 표시 형식을 정리했습니다.

- 숫자는 1,000 단위 쉼표를 적용합니다.
- 비율은 0.00% 형식으로 표시합니다.
- 광고수익률은 배수가 아니라 퍼센트 방식으로 표시합니다.
- 화면과 보고서의 주요 항목은 한글로 표시합니다.
- 주요 용어에는 영문명을 괄호로 함께 표기합니다.
- Excel 결과 파일에는 컬럼 설명 메모를 넣었습니다.
- Excel 결과 파일에는 틀고정과 필터를 적용했습니다.
- 기간별 분석 시트가 자동으로 생성되도록 구성했습니다.

## 8. 기간 분석 기능

현재 AIMAOS는 단순 전체 합계만 보는 구조가 아니라, 기간별로 성과를 나누어 볼 수 있도록 설계되어 있습니다.

현재 지원하거나 준비된 기간 기준은 다음과 같습니다.

- 일자별
- 주간별
- 월별
- 분기별
- 반기별
- 연도별
- 시즌별
- 사용자 지정 기간
- 두 기간 비교 분석

사용자 지정 기간 분석은 업로드된 데이터에 날짜 컬럼이 있을 때 사용할 수 있습니다. 날짜 컬럼이 없는 파일에서는 기간 분석 기능을 제한하는 방향이 안전합니다.

## 9. 운영 점검 포인트

기존의 "이상징후"라는 표현은 초보자에게 과하게 부정적으로 보일 수 있어, 화면에서는 "운영 점검 포인트"라는 표현으로 바꾸었습니다.

운영 점검 포인트는 광고 성과에서 확인이 필요한 항목을 알려주는 영역입니다.

예시는 다음과 같습니다.

- 광고비는 사용되었지만 전환이 부족한 캠페인
- 클릭률이 낮아 소재나 키워드 점검이 필요한 캠페인
- 광고수익률이 낮아 예산 조정이 필요한 캠페인
- 성과가 좋은데 예산 확대 검토가 가능한 캠페인

이 영역은 단순 경고가 아니라, 담당자가 무엇을 먼저 봐야 하는지 알려주는 운영 우선순위 화면입니다.

## 10. 채널 통합 관제 시스템

AIMAOS는 단순 광고매체 분석기에서 끝나지 않고, 통합 마케팅 운영 관제 시스템으로 확장되도록 구조를 추가했습니다.

채널은 DB에서 관리하며, 화면 코드에 고정하지 않는 방향으로 설계했습니다.

현재 기본 채널 분류는 다음과 같습니다.

| 영역 | 목적 | 예시 채널 |
| --- | --- | --- |
| 광고매체 | 광고 집행 성과 분석 | 네이버 검색광고, 네이버 쇼핑광고, 네이버 브랜드검색, 카카오 광고, 구글 광고, 메타 광고, 틱톡 광고, 유튜브 광고, 당근 광고 |
| 오픈마켓 | 판매채널 성과 분석 | 쿠팡, G마켓, 옥션, 11번가 |
| 커머스 플랫폼 | 성장 채널 분석 | 스마트스토어, SSG닷컴, 롯데온, 컬리, 오늘의집, 무신사, 토스쇼핑 |
| 글로벌 채널 | 해외 판매 및 신흥 채널 분석 | 알리익스프레스, 테무, 아마존, 쇼피, 큐텐 |

현재 채널 통합 관제 화면은 실제 외부 데이터를 수집하지 않습니다. 샘플 데이터를 기반으로 향후 통합 관제 화면이 어떤 구조로 확장될지 보여주는 준비 화면입니다.

## 11. 채널 관리자

채널 관리자는 향후 채널이 늘어나는 상황을 대비하기 위한 관리 화면입니다.

현재 가능한 작업은 다음과 같습니다.

- 채널 목록 확인
- 채널 추가
- 채널 수정
- 채널 활성화 또는 비활성화
- 채널 그룹 지정
- 채널 표시 색상 지정
- 분석 지표 지정
- 화면 표시 순서 지정

이 구조 덕분에 새로운 채널이 추가되어도 코드 전체를 크게 바꾸지 않고 DB에 채널을 추가하는 방식으로 확장할 수 있습니다.

## 12. Phase 2 확장 준비

현재는 실제 크롤러, 로그인 자동화, 외부 수집 기능을 구현하지 않았습니다. 대신 향후 확장을 위한 메뉴와 문서 구조를 먼저 만들었습니다.

확장 준비가 된 영역은 다음과 같습니다.

| 영역 | 향후 목표 |
| --- | --- |
| Commerce Intelligence | 스마트스토어, 쿠팡, G마켓, 옥션, 11번가 등의 상품, 가격, 리뷰, 평점, 판매량 데이터를 분석합니다. |
| Search Intelligence | 네이버, 구글, 카카오다음, 네이트의 키워드, 검색량, 경쟁도, 검색 결과를 분석합니다. |
| AI Visibility Intelligence | ChatGPT, Claude, Gemini, Perplexity에서 브랜드가 어떻게 언급되고 추천되는지 분석합니다. |
| Channel Intelligence | 광고매체, 판매채널, 커머스 플랫폼, 글로벌 채널을 통합해서 운영 성과를 봅니다. |

## 13. 현재 하지 않은 것

현재 단계에서 의도적으로 하지 않은 작업은 다음과 같습니다.

- 실제 광고센터 로그인 자동화
- 실제 크롤러 구현
- 실제 외부 사이트 데이터 수집
- 캡차 우회
- 보안 우회
- 정책 위반 가능성이 있는 자동화
- 기존 광고 데이터 분석 엔진의 대규모 변경
- 무리한 SaaS 플랫폼화

이유는 MVP의 안정성이 최우선이기 때문입니다. 현재 광고 데이터 분석기가 깨지면 AIMAOS의 첫 번째 수익화 자산이 흔들리므로, 확장 기능은 먼저 구조만 준비하고 실제 수집은 샘플과 검증 후 단계적으로 붙이는 방향이 맞습니다.

## 16. 설명용 짧은 발표 문장

AIMAOS는 광고 데이터를 자동으로 정리하고 분석해서, 광고주 보고서와 운영 점검 포인트를 만들어주는 AI마케팅협회의 운영 시스템입니다.

현재 MVP는 광고센터에서 내려받은 Excel 또는 CSV 파일을 업로드하면, 노출수, 클릭수, 광고비, 전환수, 매출, 광고수익률 같은 핵심 지표를 자동 계산하고, 초보자도 이해할 수 있는 한글 보고서를 생성합니다.

동시에 향후에는 광고매체뿐 아니라 쿠팡, 스마트스토어, G마켓, 옥션, 11번가, 글로벌 채널, 검색 결과, AI 답변 노출까지 통합 관리할 수 있도록 채널 통합 관제 구조를 준비했습니다.

즉, AIMAOS의 1단계는 광고 데이터 분석기이고, 최종 목표는 대한민국 중소기업과 광고대행사를 위한 통합 마케팅 운영 관제 플랫폼입니다.

## 17. 다음 개발 우선순위

현재 기준에서 다음 개발 우선순위는 아래 순서가 적절합니다.

1. 실제 광고주 샘플 Excel과 CSV를 더 많이 확보해서 컬럼 매핑 정확도를 높입니다.
2. 사용자 지정 기간 분석 화면을 더 보기 쉽게 개선합니다.
3. 두 기간 비교 분석 결과를 Excel 보고서에도 더 명확하게 반영합니다.
4. 채널별 수동 업로드 템플릿을 만듭니다.
5. 채널 통합 KPI 계산 모듈을 별도로 추가합니다.
6. 업종별 벤치마크 저장 구조를 만듭니다.
7. PDF 업로드는 실제 광고 매체 PDF 샘플을 받은 뒤 별도 모듈로 검토합니다.
8. 크롤러와 로그인 자동화는 정책, 안정성, 유지보수 리스크를 검토한 뒤 마지막 단계에서 제한적으로 적용합니다.

## 18. 최종 판단

현재 AIMAOS는 광고 데이터 분석 MVP로서 테스트 가능한 상태입니다. 기존 Excel/CSV 분석 기능은 핵심 자산으로 유지하고, 그 위에 기간 분석, 운영 점검 포인트, 한글 보고서, 채널 통합 관제 구조를 얹어 확장 기반을 만들었습니다.

지금 가장 중요한 것은 기능을 무작정 늘리는 것이 아니라, 실제 광고주 데이터를 넣었을 때 분석 품질이 얼마나 안정적으로 나오는지 확인하는 것입니다. 이후 검증된 데이터와 반복 운영 사례가 쌓이면, 이 시스템은 단순 자동화 도구가 아니라 AI마케팅협회의 데이터 자산, 지식 자산, 컨설팅 자산으로 확장될 수 있습니다.
